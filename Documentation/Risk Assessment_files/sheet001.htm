DevOps-Practical Risk Assessment
Potential Risk? Risk Rating? Who is at risk? Actions taken? Further Details? By Whom? When? Done?
Database is hacked and the whole database is vulnerable. Low (Security Measures in place) All Staff, Developers, Users & partners. All Data in and out of the database will be ecrypted. Variables are handled by jenkins GCP allows us for automatic sql backups at user set intervals Developers Immediately Yes
Data is lost for whatever reason. Low (Application is ran on the cloud) All Staff, Developers, Users & partners. All data will be backed up on a 12-24 hour bases. Developers & Cloud/Host Provider Immediately Yes
Communication lost Low (Application is ran on the cloud) All Staff, Developers, Users & partners. Determine location/region. Contact & check Host/Cloud Provider Ran on the cloud with multiple machines, if one goes down there is a backup Cloud Provider Immediately Yes
Developers Jenkins is Hacked Low (Security Measures in place) All Staff, Developers, Users, partners & all their data Secure passwords, managed by developers. To be hacked they must fist hack into devs computers, then access devs GCP Developers Immediately Yes
Ansible does not work. Low (Unlikely but possbile) Asible installs docker, without docker no service. Jenkins could install docker as a stage after ansible in the pipeline This wouldn’t cause duplicates, if its installed great move on if not install it Developers Immediately No
Testing Failures Medium Developers Determine why tests are failing and take actions like different methods to fix While automation of testing is easy Jenkins can be a little tempermental about how theyre called Developers Immediately Yes
Environment Variables Exposed Low (Security Measures in place) All Staff, Developers, Users, partners & all their data Secure passwords, managed by developers. The only way these variables can be exposed is if they had access to my Jenkins Developers Immediately Yes
Docker Swarm Manager goes down Low (Ran on the cloud) All Staff, Developers, Users, partners & all their data Restart the warm, determine root of problem through logs and minimise downtime If the Swarm Manager goes down so do the child nodes which means no service. Developers Immediately Yes
Swarm workers/child nodes go down Low (Ran on the cloud) All Staff, Developers, Users, partners & all their data Restart the warm, determine root of problem through logs and minimise downtime Whats wonderful is that even if one of the child nodes go down swarm will make use of whats left of remaining nodes Developers Immediately Yes
